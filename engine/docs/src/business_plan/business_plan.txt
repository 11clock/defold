Dynamo affärsplan
=================
:company:   Dynamo
:service:   Dynamo

Affärsidé
---------

// Kundbehov
{service} är en tjänst för kunder som vill skapa interaktiva digitala applikationer,
med fokus på spel för mobiltelefoner. Tjänsten innefattar verktyg för att
producera applikationen, lagring av resurser och projektadministrering.
Tjänsten erbjuds via internet och hanterar att kunder på olika platser
kan arbeta tillsammans.

// Problem för kunder
Att göra en interaktiv applikation kräver traditionellt sett bred
teknisk kunskap. {service} är utformat på ett sådant sätt att kunder som
saknar den kunskapen ändå skall kunna förverkliga sin idé. I regel krävs det
dessutom en mängd olika verktyg för alla olika arbetsmoment under
produktionen. {service} innefattar funktionalitet för alla dessa moment,
vilket sänker inlärningskurvan och eliminerar beslut av vilka olika
verktyg som skall användas.

// Vem är kunden?
Tjänsten är utformad för alla kreativa personer som vill skapa en interaktiv
applikation. Liksom alla datorbaserade verktyg krävs det givetvis en viss nivå
av datorvana för att kunna använda tjänsten. Vi har däremot fokuserat på att
göra tjänsten så enkel som möjligt att använda för att nå en bredare målgrupp
än liknande tjänster.

// Unik egenskap, specialité.
Det finns en uppsjö av liknande tjänster på marknaden. Det som saknas är
en heltäckande lösning för alla arbetsmoment. Genom erfarenhet har vi märkt att
tiden det tar från att en ändring görs tills den kan testas live i applikationen
är avgörande för den slutliga kvalitéen. Därför har vi gjort det möjligt att i
realtid se ändringar i applikationen medans den kör på målplattformen, t.ex. en
mobiltelefon.

// Kundnytta
Med {service} kan kunder med minimal teknisk kunskap billigt och enkelt
skapa interaktiva applikationer. Tjänsten uppmanar dessutom till samarbete för
användarna, även om de befinner sig på vitt skilda geografiska platser.

// Förändring
Ursprungligen ville vi göra egna spel för mobiltelefoner. Eftersom våra
krav på verktyg och tjänster inte kunde tillfredställas av det marknaden
erbjuder, började vi utveckla egna lösningar. Med tiden insåg vi att
resultatet var över förväntan och att en komplett lösning för
spelproduktion, som saknar motstycke, började ta form. Vi beslutade då att
våra verktyg var det vi skulle erbjuda till andra med samma krav.

// Särskilda egenskaper är redan täckta.

Affärsmodell
------------

// Månadsavgift
Affärsmodellen baseras framförallt på en månatlig avgift för användande
av tjänsten. Avgiften sätts per projekt och ökar med antalet deltagare i
projektet. Vi kommer också erbjuda tilläggstjänster som utökat
lagringsutrymme, felhantering och dylikt till en högre månadskostnad.

// Royalties
För lovande projekt kan vi också starta ett närmre samarbete med kunden
där vi ger extra support och eventuellt skräddarsydda lösning för
specifika problem. I gengäld kommer vi då kräva en procent av
försäljningsintäkter.

Affärssystem
------------

image:images/business_system.png[Affärssystem]

Processen börjar med att en projektägare skapar ett projekt för applikationen
som skall utvecklas. Projektägaren får då välja en licens som passar projektets omfång.
Licenserna baseras i huvudsak på hur många deltagare ett projekt tillåts ha.
Projektägaren måste därför ange betalningsinformation för att projektet skall kunna
skapas, t.ex kreditkortsnummer. En månatlig automatisk debitering mot {company}
startar omedelbart.

Projektägaren lägger därefter till deltagare (ägaren själv är givetvis redan
deltagare) till sitt projekt. Varje deltagare kan ladda hem {service}
Editor från hemsidan. Editorn är det centrala verktyger för att utveckla
applikationen. Genom editorn kopplar användaren upp sig mot det specifika
projektet hon vill arbeta med.

Efter att applikationen har nått önskvärd kvalité kan projektägaren publicera
applikationen till önskvärt köpställe, t.ex. Apple App Store eller Android Market.

Kundavtal
~~~~~~~~~

Alla registrerade kunder måste ingå i ett avtal som täcker:

* Förlust av information från företagets server
* Att ingen olaglig information lagras på företagets server
* Upphovsrätt

* Royalties
* Upphovsrätt

Belöningssystem
---------------

// Lön
// Netflix, för extremt?
